2. Integracion real del modelo hibrido con la web
------------------------------------------------

Se dejo conectado el endpoint principal de la web con el sistema hibrido final.

Archivo principal del motor:

ml_service/recommend_route.py

Este script:

- carga data/pois_barcelona_hibrido.parquet
- calcula TF-IDF sobre content_base
- calcula similarity_score
- genera candidatos hibridos
- calcula hybrid_candidate_score
- construye la ruta greedy
- devuelve candidates, route, summary y meta

El backend Node.js llama a este script desde:

project-root/backend/src/services/recommendationService.js

La funcion principal:

recommendRoute(preferences)

delegada ahora en:

runHybridRecommender(preferences)

La heuristica temporal de Node.js se conserva como referencia, pero el endpoint
principal ya usa:

mode: python-hybrid-recommender
dataset: pois_barcelona_hibrido.parquet

5. Numero minimo de POIs
------------------------

Se anadio el campo:

minPois

En el frontend:

project-root/frontend/src/components/PreferenceForm.jsx

Validaciones:

- no puede ser negativo
- debe ser menor o igual que maxPois

En el motor Python:

ml_service/recommend_route.py

Si el modelo no puede generar una ruta con el minimo solicitado, devuelve:

MIN_POIS_NOT_REACHED

El frontend lo muestra como mensaje claro:

"No se ha podido generar una ruta con el numero minimo de POIs solicitado.
Prueba a aumentar la distancia maxima o reducir el minimo de POIs."

11. Como comprobar que se usa el modelo hibrido final
----------------------------------------------------

En la respuesta del backend debe aparecer:

meta.mode = python-hybrid-recommender
meta.dataset = pois_barcelona_hibrido.parquet

Si apareciera:

node-temporary-heuristic

significaria que se esta usando la heuristica antigua.

Actualmente recommendRoute() en:

project-root/backend/src/services/recommendationService.js

llama a:

runHybridRecommender(preferences)
